🤔 为什么要学 AI Agent？

2024-2025 年，AI 领域最火的方向就是 Agent。

从 ChatGPT 到 Cursor、Devin、Manus —— AI 正在从「回答问题」进化为「自主完成任务」。

—— 这不是趋势，是正在发生的事

🛠️

自动写代码

Cursor、GitHub Copilot 帮你写代码、修 Bug、做 Code Review

🔍

智能搜索

Perplexity 自动搜索多个来源，汇总分析，给你结论

🤖

自动化助手

Manus、Devin 自动完成复杂任务，从规划到执行一条龙

💬 ChatBot

  • 你问一句，它答一句
  • 没有记忆
  • 不能使用工具
  • 不能自主行动
  • 被动响应
vs

🤖 Agent

  • 理解目标，自主规划
  • 有长期记忆
  • 能调用工具和 API
  • 能循环执行直到完成
  • 主动行动

💡 一句话理解 Agent

Agent = LLM + Tools + Memory + Planning

大语言模型是「大脑」，工具是「手脚」，记忆是「笔记本」，规划是「思考方式」。

这四个组件缺一不可！

🧠
组件 1

LLM — 大语言模型

Agent 的「大脑」。负责理解用户意图、推理决策、生成回复。

常见的 LLM：GPT-4、Claude、Llama、Qwen

它不只是生成文字，更重要的是做决策。

LLM 是 Agent 的核心引擎

🛠️
组件 2

Tools — 工具

Agent 的「手脚」。LLM 本身只能生成文字，工具让它能：

  • 🔍 搜索网页
  • 💻 执行代码
  • 📁 读写文件
  • 🌐 调用 API

Function Calling 是关键技术

📝
组件 3

Memory — 记忆

Agent 的「笔记本」。让 Agent 记住：

  • 短期记忆：当前对话上下文
  • 长期记忆：用户偏好、历史经验
  • 工作记忆：当前任务状态

没有记忆的 Agent 是金鱼 🐟

🗺️
组件 4

Planning — 规划

Agent 的「思考方式」。包括：

  • 任务分解：大任务拆成小步骤
  • 反思纠错：执行后检查结果
  • 策略选择：选最优路径

常用技术：ReAct、Chain-of-Thought

先想后做，边做边改

🧠

LLM

大脑：理解与决策

🛠️

Tools

手脚：执行操作

📝

Memory

笔记本：记住经验

🗺️

Planning

思维：规划路径

四大组件协同工作，缺一不可！

🔄 Agent Loop — 核心循环

Agent 的工作流程是一个循环：

① 接收任务 → ② 思考规划 → ③ 选择工具 → ④ 执行动作 → ⑤ 观察结果 → ⑥ 判断是否完成

如果没完成 → 回到第②步继续

这就是 ReAct: Reason + Act

Agent Loop 流程

关键步骤

Reason思考
Act行动
Observe观察
Repeat重复

循环直到任务完成！

📦 传统程序

  • 流程写死（if-else）
  • 输入固定格式
  • 不能处理意外
  • 每个功能都要手写
  • 修改需要重新编程
vs

🤖 AI Agent

  • 动态决策（LLM 推理）
  • 自然语言输入
  • 灵活应对变化
  • 通过工具自动扩展能力
  • 用 prompt 就能调整行为

📞 Function Calling

Agent 调用工具的核心机制：

1️⃣ 告诉 LLM 有哪些工具可用（函数定义）

2️⃣ LLM 分析请求，决定调用哪个工具

3️⃣ LLM 生成调用参数（JSON）

4️⃣ 代码执行工具，返回结果给 LLM

5️⃣ LLM 基于结果继续思考或回复

LLM 与外界交互的桥梁！

Step 1

初始化

创建 OpenAI client，设置 system prompt 和 tools 定义

Step 2

循环

while True: 发消息 → 检查 tool_calls → 执行 → 追加结果

Step 3

输出

没有 tool_calls → 循环结束 → 返回最终回复

核心伪代码 
messages = [system_prompt, user_msg]

while True:
    response = llm.chat(messages, tools)

    if response.tool_calls:
        for call in tool_calls:
            result = execute(call)
            messages.append(result)
    else:
        print(response.content)
        break
要点

核心就三件事

  • 📨 发消息给 LLM
  • 🔧 处理 tool_calls
  • 🔁 循环到结束
