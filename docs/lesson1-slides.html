🧠 架构设计：LLM + Harness

🧠 LLM（大脑）

  • 理解需求、推理规划
  • 决定下一步做什么
  • 解析多种编程语言
  • 将复杂任务拆分为步骤

🦴 Harness（骨架）

  • 工具注册 — 告诉 LLM 有哪些工具
  • 工具执行 — LLM 说调什么就真的去调
  • 上下文管理 — 对话历史、文件内容
  • 循环控制 — 何时继续、何时停
  • 安全边界 — 权限控制、变更回滚

没有工具，LLM 只能生成文本
有了工具 + 循环，它才变成 Agent 🚀

🔧 Harness 的核心：工具系统

📁

文件操作

读取、修改、创建文件

🔍

搜索

按模式查找文件
正则搜索内容

⚡

命令执行

Shell、测试
Git、构建

🌐

Web

搜索网页
查文档

每次工具调用都产生新信息 → 反馈给模型 → 指导下一步决策
这就是 Agentic Loop 的运行过程 💡

🧩 Agent Loop 需要哪些部分？

def agent_loop(user_message): messages = [system_prompt, user_message] while True: response = call_llm(messages) # 1 if has_tool_calls(response): # 2 result = execute_tools(...) # 3 messages.append(result) # 4 else: return response.content # 5

1. System Prompt — 定义角色和工具

2. 消息历史 — 维护上下文

3. LLM 调用 — 请求模型

4. 工具分发 — 解析 tool_calls，路由执行

5. 循环判断 — 有工具调用就继续，否则返回
